Thank you for your interest in contributing to wa_direct_chat!

Getting started
---------------

1. Fork the repository on GitHub.
2. Create a feature branch: `git checkout -b feat/my-feature`.
3. Make changes and run formatting/tests locally.

Local setup
-----------

```bash
flutter pub get
flutter format .
flutter analyze
flutter test
```

Code style
----------

- Follow Dart/Flutter style using `flutter format`.
- Keep changes small and focused per PR.

Commit messages
---------------

- Use present tense, short summary, and optional longer description.
- Example: `feat: add phone number validation`.

Pull request process
--------------------

- Open a PR against the `main` branch.
- Describe the change, include screenshots if UI-related, and list any breaking changes.
- Link related issues or include reproduction steps.

Reporting bugs
--------------

- Open an issue with a clear title and reproduction steps.

Code of conduct
---------------

This project follows the [Contributor Covenant](CODE_OF_CONDUCT.md). By participating, you agree to abide by its terms.

Thanks for helping improve wa_direct_chat!
